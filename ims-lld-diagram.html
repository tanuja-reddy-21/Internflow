Request Lifecycle & JWT Middleware Chain
1
HTTP Request arrives
Client sends: Authorization: Bearer <token>
2
cors() + express.json()
Global Express middleware applied
3
verifyToken middleware
jwt.verify(token, JWT_SECRET) → attaches req.user {id, role}
4
authorizeRole(...roles) middleware
Check req.user.role ∈ allowedRoles → 403 if denied
5
Route Handler → Controller
Business logic executes → calls Service layer
6
Service → MongoDB via Mongoose
DB operation → response JSON → client

JWT Auth Flow
Register
1. Validate email uniqueness
2. bcrypt.hash(password, 10)
3. Save User to MongoDB
4. Return 201 Created
Login
1. Find user by email
2. bcrypt.compare(input, hash)
3. jwt.sign({id, role}, SECRET, {expiresIn:'7d'})
4. Return token to client
Token Validation
Extract: req.headers.authorization
Split: "Bearer <token>"
Verify: jwt.verify(token, SECRET)
Attach: req.user = decoded payload
Invalid → 401 Unauthorized
Attendance Check-In Guard
1. Get req.user.id from token
2. today = new Date().setHours(0,0,0,0)
3. Query: Attendance.findOne({internId, date:today})
4. If found → 409 Already checked in
5. Else → Create new Attendance record
6. Return 201 with record
End-to-End Data Flow — Example: Intern Submits Task
① React Client
User clicks "Submit Task"
SubmitTask.jsx calls taskAPI.submit(id, link)
axios sends POST /api/tasks/:id/submit + Bearer token
② Middleware
verifyToken() extracts & validates JWT
req.user = {id: internId, role: 'intern'}
authorizeRole('intern') passes ✓
③ Controller / Service
taskController.submitTask(req, res)
Validates req.user.id === task.assignedTo
taskService.updateSubmission(id, link)
④ Database
Task.findByIdAndUpdate(id, {submissionLink, status:'completed'})
Mongoose returns updated document
⑤ Response
Controller sends 200 + updated task JSON
React state updated → TaskList re-renders
PerformanceCard recalculates metrics
